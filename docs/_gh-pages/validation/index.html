Validation¶

Validation happens all over the place. These are validation functions that ended up in the same file.

Domain/ReverseDomain Validation¶

Use the function do_zone_validation in model code.

mozdns.validation.do_zone_validation(domain)¶

Preform validation on domain. This function calls the following functions:

check_for_soa_partition
check_for_master_delegation
validate_zone_soa

Note

The type of the domain that is passed is determined dynamically

Parameters:domain (Domain or ReverseDomain) – The domain/reverse_domain being validated.

The following code is an example of how to call this function during domain introspection.

>>> do_zone_validation(self, self.master_domain)

The following code is an example of how to call this function during reverse_domain introspection.

>>> do_zone_validation(self, self.master_reverse_domain)
mozdns.validation.check_for_master_delegation(domain, master_domain)¶

No subdomains can be created under a domain that is delegated. This function checks whether a domain is violating that condition.

Parameters:
  • domain (Domain or ReverseDomain) – The domain/reverse_domain being validated.
  • master_domain (Domain or ReverseDomain) – The master domain/reverse_domain of the domain/reverse_domain being validated.

The following code is an example of how to call this function during domain introspection.

>>> check_for_master_delegation(self, self.master_domain)

The following code is an example of how to call this function during reverse_domain introspection.

>>> check_for_master_delegation(self, self.master_reverse_domain)
mozdns.validation.validate_zone_soa(domain, master_domain)¶

Make sure the SOA assigned to this domain is the correct SOA for this domain. Also make sure that the SOA is not used in a different zone.

Parameters:
  • domain (Domain or ReverseDomain) – The domain/reverse_domain being validated.
  • master_domain (Domain or ReverseDomain) – The master domain/reverse_domain of the domain/reverse_domain being validated.

The following code is an example of how to call this function during domain introspection.

>>> validate_zone_soa('forward', self, self.master_domain)

The following code is an example of how to call this function during reverse_domain introspection.

>>> validate_zone_soa('reverse', self, self.master_reverse_domain)
mozdns.validation.check_for_soa_partition(domain, child_domains)¶

This function determines if changing your soa causes sub domains to become their own zones and if those zones share a common SOA (not allowed).

Parameters:
  • domain (Domain or ReverseDomain) – The domain/reverse_domain being validated.
  • child_domains (Domain or ReverseDomain) – A Queryset containing child objects of the Domain/ReverseDomain object.
Raises :

ValidationError

The following code is an example of how to call this function during domain introspection.

>>> check_for_soa_partition(self, self.domain_set.all())

The following code is an example of how to call this function during reverse_domain introspection.

>>> check_for_soa_partition(self, self.reversedomain_set.all())
mozdns.validation.find_root_domain(soa)¶

It is nessicary to know which domain is at the top of a zone. This function returns that domain.

Parameters:soa (SOA) – A zone’s SOA object.

The following code is an example of how to call this function using a Domain as domain.

>>> find_root_domain('forward', domain.soa)

The following code is an example of how to call this function using a ReverseDomain as domain.

>>> find_root_domain('reverse', reverse_domain.soa)

Name and Label Validation¶

mozdns.validation.validate_label(label, valid_chars=None)¶

Validate a label.

Parameters:label (str) – The label to be tested.

“Allowable characters in a label for a host name are only ASCII letters, digits, and the ‘-‘ character.”

“Labels may not be all numbers, but may have a leading digit”

“Labels must end and begin only with a letter or digit”

– RFC

“[T]he following characters are recommended for use in a host name: “A-Z”, “a-z”, “0-9”, dash and underscore”

– RFC

mozdns.validation.validate_domain_name(name)¶

Domain names are different. They are allowed to have ‘_’ in them.

Parameters:name (str) – The domain name to be tested.
mozdns.validation.validate_name(fqdn)¶

Run test on a name to make sure that the new name is constructed with valid syntax.

Parameters:fqdn (str) – The fqdn to be tested.

“DNS domain names consist of “labels” separated by single dots.” – RFC

Note

DNS name hostname grammar:

<domain> ::= <subdomain> | " "

<subdomain> ::= <label> | <subdomain> "." <label>

<label> ::= <letter> [ [ <ldh-str> ] <let-dig> ]

<ldh-str> ::= <let-dig-hyp> | <let-dig-hyp> <ldh-str>

<let-dig-hyp> ::= <let-dig> | "-"

<let-dig> ::= <letter> | <digit>

<letter> ::= any one of the 52 alphabetic characters A
through Z in upper case and a through z in lower case

<digit> ::= any one of the ten digits 0 through 9

–RFC 1034

mozdns.validation.validate_reverse_name(reverse_name, ip_type)¶

Validate a reverse name to make sure that the name is constructed with valid syntax.

Parameters:reverse_name (str) – The reverse name to be tested.

SRV Validation¶

SRV objects need special validation because of the _ the precedes their names. They also have other fields like weight, port and priority that need to be validated.

mozdns.validation.validate_srv_port(port)¶

Port must be within the 0 to 65535 range.

mozdns.validation.validate_srv_priority(priority)¶

Priority must be within the 0 to 65535 range.

mozdns.validation.validate_srv_weight(weight)¶

Weight must be within the 0 to 65535 range.

mozdns.validation.validate_srv_label(srv_label)¶

This function is the same as validate_label() expect SRV records can have a _ preceding its label.

mozdns.validation.validate_srv_name(srv_name)¶

This function is the same as validate_name() expect SRV records can have a _ preceding is name.

MX Validation¶

mozdns.validation.validate_mx_priority(priority)¶

Priority must be within the 0 to 65535 range.

IP type Validation¶

mozdns.validation.validate_ip_type(ip_type)¶

An ip_type field must be either ‘4’ or ‘6’.
